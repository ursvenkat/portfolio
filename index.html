About

I'm a full stack developer with three-plus years of experience designing, building, and supporting enterprise web applications — most of it spent inside Wipro's FedEx account, where I built REST APIs and microservices and kept production systems running under SLA.

Today I work on iHub, a logistics automation platform built for APL Logistics inside Kintetsu World Express. I lead a data migration framework called DMFW that moves invoice and shipment records out of SAP and into PostgreSQL correctly — staged, idempotent, and auditable — and I build the n8n workflows that keep freight and invoice data moving without anyone babysitting them.

My foundation is Java and Spring Boot, sharpened over years of production support where root-cause analysis and clean incident response mattered as much as writing the code in the first place. I've since built outward into JavaScript, Python, and workflow automation tooling, while keeping that same backend discipline: correct before clever.

YINCIO001 — Invoice Migration Pipeline

The centerpiece of my DMFW work: an SAP-to-PostgreSQL migration for invoice data. It extracts through a chain of CTEs, stages records idempotently, and posts atomically into six production tables — currently being refactored so the posting node fetches in bulk and prepares data in pure JavaScript, instead of hitting the database once per invoice.
